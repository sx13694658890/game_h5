## 一、项目概述

### 1.1 游戏定位

一款轻量化、无需安装、开箱即玩的 H5 弹球休闲游戏。玩家通过触摸/点击控制挡板，接住下落的星球弹球，消除砖块获得分数。游戏具有星际科幻美学风格，视觉层次丰富、操作简单易上手。

### 1.2 核心玩法

- 玩家滑动底部挡板左右移动
- 弹球在屏幕内弹射，碰撞砖块消除得分
- 砖块全部消除后进入下一关，难度递增
- 球落底则失去一条命，三命后游戏结束

### 1.3 设计风格

| 维度 | 选择 |
|------|------|
| 主题 | 深空星际 · 暗色系 |
| 主色调 | 深蓝黑 `#0a0e1a` + 星光金 `#f7c948` + 霓虹青 `#00e5ff` |
| 字体 | 标题：Orbitron（等宽科技感）/ 正文：DM Sans |
| 动效 | 粒子拖尾、砖块爆炸、关卡过渡光效 |
| 风格关键词 | 赛博、极简、沉浸 |

### 3.4 游戏循环

```
requestAnimationFrame Loop (60fps)
  │
  ├─ 1. 读取输入（挡板位置）
  ├─ 2. 更新物理（球位移 = 速度 × Δt）
  ├─ 3. 碰撞检测（球 vs 墙/挡板/砖块）
  ├─ 4. 处理碰撞结果（反弹/消砖/得分/失命）
  ├─ 5. 更新粒子系统
  ├─ 6. 检测游戏状态（关卡完成/游戏结束）
  └─ 7. 渲染（Canvas 2D API 或 DOM）
```

## 五、渲染方案

### 5.1 混合渲染策略

```
┌──────────────────────────────────┐
│  React DOM 层（UI）               │  ← HUD、弹窗、菜单页面
│  ┌──────────────────────────────┐ │
│  │  Canvas 2D 层（游戏实体）     │ │  ← 球、挡板、砖块、粒子
│  └──────────────────────────────┘ │
│  CSS 层（背景星空）               │  ← 纯 CSS 动画星点
└──────────────────────────────────┘
```

**选用 Canvas 的原因**：游戏实体数量多、每帧需要大量重绘，Canvas 2D 性能优于 DOM 操作。

## 八、核心算法说明

### 8.1 砖块碰撞检测（AABB + Circle）

```
1. 找到矩形上离圆心最近的点 (cx, cy)
2. 计算距离 d = √((px-cx)² + (py-cy)²)
3. 若 d < radius → 碰撞发生
4. 判断碰撞方向（水平边 or 垂直边）→ 决定反转 vx or vy
```

### 8.2 挡板偏转算法

```
hitPos = (ballCenterX - paddleLeft) / paddleWidth  // 0.0 ~ 1.0
angle  = hitPos × 150° - 75°                       // -75° ~ +75°
vx = speed × sin(angle)
vy = -speed × cos(angle)
```

### 8.3 粒子系统

```
消砖时，生成 12~20 个粒子：
  - 初始位置：砖块中心
  - 初始速度：随机方向，3~8 px/frame
  - 生命周期：30~50 帧
  - 每帧：pos += vel, vel.y += 0.2(重力), alpha -= 1/lifetime
  - 渲染：按 alpha 绘制彩色小圆点
```

## 九、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| iOS Safari Canvas 性能不足 | 中 | 高 | 降低粒子数量；提供"简洁模式" |

| 后台标签页导致游戏异常 | 低 | 低 | `visibilitychange` 事件自动暂停 |
