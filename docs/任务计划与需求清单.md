## 二、需求清单（全量）

### 2.1 功能需求

#### FR-01 启动加载页

- **描述**：游戏首次打开时显示加载页，展示 Logo 动画与进度
- **验收标准**：
  - Logo「StarPinball」文字从透明淡入，伴随星光粒子效果
  - 进度条平滑从 0% 到 100%（预加载字体/音效资源）
  - 加载完成后自动跳转主菜单，无需用户操作
  - 加载时间超过 5s 显示"网络较慢，请稍候"提示

#### FR-02 主菜单页

- **描述**：游戏入口页面，提供导航功能
- **验收标准**：
  - 显示游戏 Logo、最高分记录
  - 包含按钮：「开始游戏」「排行榜」「设置」
  - 背景星空动画持续播放
  - 首次进入有新手提示浮层（可关闭）

#### FR-03 游戏核心场景

- **描述**：弹球游戏主玩法
- **子需求**：

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-03-01 | 弹球按物理规则在画布内运动 | P0 |
| FR-03-02 | 玩家触摸/鼠标控制挡板左右移动 | P0 |
| FR-03-03 | 球与挡板碰撞后按偏转算法反弹 | P0 |
| FR-03-04 | 球与砖块碰撞后砖块 HP 减少，归零消除 | P0 |
| FR-03-05 | 消砖得分，分值：普通=10，硬砖=20，金砖=50 | P0 |
| FR-03-06 | 连续消砖触发连击，分数乘以连击系数（最高×5） | P1 |
| FR-03-07 | 球落底扣除 1 条命，并重置球位置到挡板上方 | P0 |
| FR-03-08 | 生命值为 0 触发游戏结束流程 | P0 |
| FR-03-09 | 砖块全部消除触发关卡完成流程 | P0 |
| FR-03-10 | HUD 实时显示：当前分数、生命值、关卡数 | P0 |
| FR-03-11 | 右上角暂停按钮，点击弹出暂停弹窗 | P1 |

#### FR-04 关卡系统

- **描述**：多关卡进度管理
- **验收标准**：
  - 至少提供 5 个预设关卡配置
  - 每关砖块布局不同，难度递增（见架构文档 §4.2）
  - 关卡信息持久化，刷新后可从当前关卡继续（P2）

#### FR-05 粒子特效

- **描述**：视觉反馈特效
- **验收标准**：
  - 砖块消除时在砖块位置产生 12~20 个彩色粒子散射
  - 粒子有重力效果，生命周期内透明度渐变至消失
  - 弹球移动时有 5 帧拖尾残影
  - 金砖消除时粒子颜色为金色，效果更强烈

#### FR-06 音效系统

- **描述**：游戏音频反馈
- **验收标准**：
  - 球与挡板碰撞：短促"嘭"声
  - 球与砖块碰撞：清脆"叮"声
  - 砖块消除：爆裂音效
  - 失去一条命：低沉警示音
  - 关卡完成：胜利旋律（3秒）
  - 游戏结束：失败音效
  - 所有音效由 Web Audio API 合成，无需加载音频文件

#### FR-07 暂停弹窗

- **描述**：游戏中途暂停功能
- **验收标准**：
  - 暂停后游戏循环完全停止（球不移动）
  - 弹窗显示：「继续游戏」「重新开始」「返回主菜单」
  - 点击继续后游戏从暂停位置恢复

#### FR-08 关卡完成弹窗

- **描述**：关卡通关后展示
- **验收标准**：
  - 显示本关得分、星级评定（1~3星，按清除时间和剩余生命数判定）
  - 「下一关」按钮进入下一关卡
  - 星级动画依次出现（非同时显示）

#### FR-09 游戏结束页

- **描述**：所有生命耗尽后的结算页
- **验收标准**：
  - 显示最终得分、到达关卡、本次是否刷新最高分
  - 刷新最高分时有特殊动效（金色光效+文字"NEW RECORD!"）
  - 按钮：「再玩一次」「返回主菜单」
  - 自动将分数写入本地排行榜

#### FR-10 本地排行榜

- **描述**：保存历史成绩
- **验收标准**：
  - 存储最近 10 条记录（分数、关卡、日期）
  - 按分数从高到低排列
  - 当前游戏记录高亮显示
  - 支持清空排行榜（需二次确认）

#### FR-11 设置页

- **描述**：用户偏好设置
- **验收标准**：
  - 音效开关（默认开启）
  - 难度选择：简单 / 普通 / 困难（影响球速和挡板宽度的基础值）
  - 设置项存入 localStorage，下次打开自动恢复

### 2.2 非功能需求

| 需求 ID | 类型 | 描述 | 指标 |
|---------|------|------|------|
| NFR-01 | 性能 | 游戏帧率 | ≥ 60fps（中端手机） |
| NFR-02 | 性能 | 首屏加载 | ≤ 2s（4G） |

| NFR-08 | 可靠性 | 后台自动暂停 | 切后台 100% 触发 |

## 五、需求步骤详解（Step-by-Step）

### Step 1：环境搭建

```bash
# 1. 创建项目
npm create vite@latest star-pinball -- --template react-ts
cd star-pinball

# 2. 安装依赖
npm install zustand framer-motion
npm install -D tailwindcss autoprefixer postcss

# 3. 初始化 Tailwind
npx tailwindcss init -p

# 4. 安装开发工具
npm install -D eslint prettier eslint-plugin-react-hooks

# 5. 启动验证
npm run dev
```

### Step 2：全局样式配置

```css
/* src/index.css */
:root {
  --color-bg:        #0a0e1a;
  --color-primary:   #00e5ff;   /* 霓虹青 */
  --color-accent:    #f7c948;   /* 星光金 */
  --color-danger:    #ff4757;   /* 危险红 */
  --color-surface:   #111827;   /* 卡片背景 */
  --color-border:    #1e293b;

  --font-display:    'Orbitron', sans-serif;
  --font-body:       'DM Sans', sans-serif;

  --radius-brick:    4px;
```

### Step 4：物理计算实现步骤

```
每帧执行顺序：

1. 更新球位置
   ball.pos.x += ball.vel.x
   ball.pos.y += ball.vel.y

2. 检测左右墙壁
   if (ball.pos.x - r < 0 || ball.pos.x + r > W)
     ball.vel.x *= -1

3. 检测上墙壁
   if (ball.pos.y - r < 0)
     ball.vel.y *= -1

4. 检测下边界（失命）
   if (ball.pos.y - r > H)
     → loseLife()

5. 检测挡板碰撞
   if (circleRectCollide(ball, paddle))
     → reflectOffPaddle()

6. 遍历所有砖块，检测碰撞
   for brick of bricks:
     result = circleRectCollide(ball, brick)
     if (result.hit)
       → brick.hp--
       → if (brick.hp <= 0) destroyBrick(brick.id)
       → reflectBall(result.normal)
       → break（每帧最多碰一块砖）
```
