## 1. 目标与验收标准

### 1.1 产品目标（对齐 TASK-012 / FR-02）

| 编号 | 需求来源 | 描述 | 验收要点 |
|:----:|----------|------|----------|
| G1 | TASK-012 | 随机星点闪烁 | 星点亮度/尺度变化自然，无明显同步「齐闪」 |
| G2 | TASK-012 | 多层视差星云（远 / 中 / 近） | 至少三层可区分内容，**水平漂移速度**远 < 中 < 近（或等效视差） |
| G3 | TASK-012 | 流星随机划过 | 平均 **5～10s** 出现一条；轨迹短、不挡 UI；与星点层视觉协调 |
| G4 | 阶段三 Review #3 | 不卡顿 | 中低端机主线程占用低；与游戏 Canvas **不争抢**同一绘制管线 |
| G5 | FR-02 | 主菜单背景 | 与现有 `StarBackground` 挂载位置一致，**全屏铺底**、不拦截点击（`pointer-events: none`） |

### 1.2 非目标（本版不做或降级）

- 不在 **Canvas 游戏层**内绘制星空（避免与球/粒子同画布重绘，违背架构「CSS 背景层」分工）。
- 不做需要 **WebGL** 的体积星云或复杂粒子场（成本与包体、兼容性不匹配本项目定位）。
- **加载页专属**粒子与 Logo 合成动画归 FR-01 / `LoadingPage`，本方案仅覆盖 **全局背景** `StarBackground`。

## 2. 技术路线定案

### 2.1 选型结论：**纯 CSS + 少量 DOM**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **CSS + DOM 星点** | 与架构一致、GPU 友好（`transform`/`opacity`）、易维护 | 流星用 DOM 需控制数量 | **采用** |
| Canvas 全屏背景第二画布 | 流星/粒子灵活 | 多一层 RAF 或与游戏帧率耦合、移动端耗电 | **不采用** |
| 视频 / Lottie | 效果好 | 体积大、需资源管线 | **不采用** |

**依据**：架构文档 §5.1 明确「CSS 层（背景星空）← 纯 CSS 动画星点」；游戏主体已在 Canvas，背景应保持 **独立合成层**（浏览器可对 `transform`/`opacity` 做合成）。

### 2.2 渲染层级（从下到上）

```
z-index（相对 App 内背景容器）
┌─────────────────────────────────────┐
│ L0 基底渐变（已有：深蓝 + 微弱青色/金色光晕）     │  static
├─────────────────────────────────────┤
│ L1 远层「星云」大色块 + 慢速 translateX        │  视差最慢
├─────────────────────────────────────┤
│ L2 中层「星雾」中等渐变 + 中速 translateX       │  视差中等
├─────────────────────────────────────┤
│ L3 近层「密星」小星点 + twinkle（现有加强）      │  略快或尺寸略大
├─────────────────────────────────────┤
│ L4 流星（单 DOM 元素复用，opacity+transform）   │  偶发，最顶层背景内
└─────────────────────────────────────┘
```

所有层置于现有 `StarBackground` 根节点内，保持 **`aria-hidden`**、**`pointer-events: none`**。

## 3. 分层设计参数（建议默认值，可配置常量）

> 下列数值为 **实现时的初始建议**，开发中可在 `src/config/theme.ts` 或 `StarBackground` 旁新增 `starfieldConfig.ts` 集中定义。

### 3.1 远 / 中 / 近星云（CSS）

| 层 | 表现 | 动画 | 周期（建议） |
|:--:|------|------|----------------|
| 远 | 大面积低对比径向渐变椭圆 1～2 个 | `translateX` 往返或单向循环 | **120s～180s** 一整周期 |
| 中 | 较小椭圆 + 略高透明度 | `translateX`，方向可与远层相反 | **80s～120s** |
| 近 | 弱噪点感可用 `radial-gradient` 叠 2 层 + 轻微 `scale` 呼吸 | `translateX` + 可选 `opacity` 微摆 | **45s～75s** |

**视差原则**：远层单位时间位移 **像素/秒** 最小，近层最大；三层 **勿同速同相**，避免摩尔纹式闪烁。

### 3.2 星点（DOM + CSS）

| 项 | 建议 |
|----|------|
| 数量 | **80～120** 颗（较现状 56 略增，仍属轻量）；可按 `prefers-reduced-motion` 减半 |
| 分布 | `left/top` 用固定种子随机（见 §4），避免每次热更新全屏跳动 |
| 大小 | 0.5px～2.5px 直径类圆；约 **10%** 略大作为「亮星」 |
| 闪烁 | 沿用 `@keyframes star-twinkle`，**duration** 2.5s～6s **错开**，**delay** 0～5s 随机 |

### 3.3 流星（DOM 单实例复用）

| 项 | 建议 |
|----|------|
| 触发 | `setInterval` 或 `setTimeout` 递归，间隔 **`random(5000, 10000)` ms** |
| DOM | **1 个** `<span>` 或伪元素，用 class 切换动画；避免同时多条流星 |
| 轨迹 | 对角线 `translate` + `rotate(-35deg)` 类；起点在视口外上沿随机 X |
| 时长 | 单条 **0.6s～1.2s**；`opacity` 头尾渐隐 |
| 颜色 | 主 `rgba(255,255,255,0.85)`，尾可用 `linear-gradient` 或 `box-shadow` 模拟 |

**可见性**：流星层 `z-index` 高于星点、低于 `MobileFrame` 内 **z-10 业务内容**（与当前背景在内容下一致）。

## 4. 随机与稳定性

### 4.1 伪随机种子（推荐）

- 使用 **固定种子**（如 `STARFIELD_SEED = 20260415`）的简易 LCG / mulberry32，在 `useMemo` 内生成所有星的 `left/top/size/delay/duration`，保证 **同一会话内** HMR 与路由切换不剧烈跳变。
- 若需「每次冷启动不同星空」，可改为 `sessionStorage` 存一次种子，**本方案默认**：固定种子优先，便于调试与验收录屏对比。

### 4.2 `prefers-reduced-motion`

- 系统开启「减少动态效果」时：**关闭**流星定时器；星点闪烁改为 **静态低透明度** 或极慢单次过渡；视差层 **停止** `translate` 动画或时长 ×10。

---

## 5. 性能与兼容

| 项 | 要求 |
|----|------|
| 合成 | 动画尽量只动 **`transform` / `opacity`**，避免每帧改 `top/left` 或 `box-shadow` 大范围重绘 |
| 流星 | 同时仅 **1** 条；动画结束后再调度下一次 |
| 低端机 | 提供常量 `STARFIELD_LITE = true` 或通过 `matchMedia('(max-resolution: 2dppx)')` 等简化层数（可选二期） |
| 安全区 | 背景仍 `absolute inset-0`，不受 notch 裁切问题（由外层 `MobileFrame` padding 处理） |

## 6. 文件与接口约定

| 路径 | 职责 |
|------|------|
| `src/components/ui/StarBackground.tsx` | 组装 L0～L4；流星逻辑与 `reduced-motion` 分支 |

**不修改**：`GameCanvas.tsx`（游戏层与背景解耦）。

## 7. 实现步骤（建议顺序）

1. 抽出 **L1～L2 星云** div + CSS 动画（无流星），联调透明度与主题色（`theme` / CSS 变量）。
2. 调整 **L3 星点**：数量、种子随机、与里程碑现有 `star-twinkle` 合并文档。
3. 增加 **L4 流星**：单 DOM + 动画 class + 5～10s 调度；接 `reduced-motion` 与卸载清理。
4. **真机抽检**：菜单页静置 2 分钟，观察掉帧与发热；Chrome Performance 抽样一帧长任务。
5. 更新 `docs/项目里程碑.md` 阶段三 #3 为 ✅ 或保留 🔶（若流星/视差未全达标则标 🔶 并备注）。

## 8. 验收检查清单（自测打勾）

- [ ] 远/中/近三层漂移肉眼可区分，且不同步「整块平移」。
- [ ] 星点闪烁错频，无规律性齐闪。
- [ ] 流星平均间隔在 5～10s 量级（录屏 60s 内至少 4 次为宜）。
- [ ] 菜单、游戏、排行榜、设置等路由下背景均正常（`App` 结构不变）。
- [ ] 触摸与点击不被背景拦截。
- [ ] 开启系统「减少动态效果」时无快速流星、视差显著减弱或静止。
- [ ] 与游戏同时进行时，主观无卡顿（与阶段四 60fps 联测时可合并记录）。
